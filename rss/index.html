  • Now unzip into a ghost folder or whatever folder you want to launch Ghost from.

unzip ghost-latest.zip

  • Install Ghost using npm.  You'll need to install node, per your operating system, if you haven't already and it includes npm.

npm install --production

  • You should be able to run Ghost if all went well with the previous steps.

npm start

]]>mikedevs.com/child-classes-in-arraylists/61c361c112034ec47963444fWed, 22 Dec 2021 17:57:25 GMTJava Arrays, to me, perform like any other languages array so there’s nothing exciting to say about them.  ArrayLists on the other hand are when things can get whacky-cool.  I’ll go into child classes in ArrayLists - even whackier-cooler.

Arrays are fixed in length as in other languages - booooring.  Most of what I’ve encountered irl are ArrayLists.  They are dynamic; meaning you can add and remove from them and can even take in your child classes to make things “extra”.  We can put those child classes into ArrayLists because of polymorphism in Java (that’s another cool subject).

ArrayLists are easy enough but I want to talk briefly about child classes in ArrayLists.  When you have a parent class and 1 or more child classes that extend that parent class then you can instantiate those in an array / arraylist.  It seems like an “expensive” setup, but you’re able to loop through the array to call the parent class’ methods for each child class you added to the array.  I can’t say I’ve seen this irl just yet but let’s go through a simple one I came across in Codecademy recently.

Background info

  • Language - our main/parent class
    • Constructor accepts a few items for a language
    • Has a getInfo() method that prints out basic info about a language provided to the constructor
  • Mayan / SinoTibetan - child classes
    • @Override the getInfo() method because these languages are a bit different
    • We have if statements, etc for these languages to provide other info

The Setup

  • Create the Language variables for the different child classes you will loop through

    • Language english, mayan, sino1, sino2;
      • sino2 - different word orders if does or does not contain “Chinese” in the language name
    • Instantiate each variable
    english = new Language(“English”, 1_900_999_999, “America”, “subject-verb-object”);
mayan = new Mayan(Ki’che”, 2_888_566);
sino1 = new SinoTibetan(“Mandarin Chinese”, 5_999_999);
sino2 = new SinoTibetan(“Burmese”, 6_878_878);
    • Set up the array
      • Language[] languages = {english, mayan, sino1, sino2};
    • Now loop through each language in languages to call the getInfo() method to print some information about each language. Note: English does not have a child class so it will use what’s already in the parent class for its info.
    for (Language language : languages) {
  language.getInfo();
}

We have instantiated 4 languages (child classes) from the parent (Language) in an ArrayList.  We then loop through that list to call the getInfo() method for each.  All except English will print different information because we have overriden the getInfo() method in those child classes.

Run our Language program and blam-o - we get a print out of each language’s information.
